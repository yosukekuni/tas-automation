## 2. クラウドバックアップ設計

### 2.1 Hyper Backup vs Cloud Sync 使い分け

| 比較項目 | Hyper Backup | Cloud Sync |
|----------|-------------|------------|
| 用途 | **災害復旧用バックアップ** | **日常的なファイル同期** |
| バージョン管理 | あり（世代管理・ローテーション） | なし（最新版のみ） |
| 重複排除 | ブロックレベル重複排除 | なし |
| 暗号化 | AES-256 クライアント側暗号化 | AES-256 クライアント側暗号化 |
| 圧縮 | あり（転送量・容量削減） | なし |
| 復元 | NAS全体/フォルダ/ファイル単位で復元可 | 同期のみ（削除も同期される） |
| スケジュール | 柔軟（日次/週次/カスタム） | リアルタイム/スケジュール |
| 対応先 | S3, C2, Google Drive, ローカル等 | Google Drive, OneDrive, Dropbox, S3等 |

**結論:**
- **Hyper Backup → 災害復旧用（必須）** — 世代管理・重複排除で効率的
- **Cloud Sync → 日常作業用（任意）** — リアルタイム同期が必要な場合のみ

### 2.3 推奨構成

**プライマリ: Synology C2 Storage（Basic 1TB $59.99/年）**

理由:
- Hyper Backupとの統合が最もシームレス
- 転送・復元料金なし（予測可能なコスト）
- DSMから直接復元可能
- AES-256暗号化標準

**代替案: Google Drive（Google One 2TB）**

既存Google Workspaceがあれば追加コストなしで利用可能。Hyper Backupから直接バックアップ対応。

## 3. 外部アクセス設計

### 3.1 アクセス方式の比較

| 比較項目 | QuickConnect | Tailscale VPN | DDNS + ポート開放 |
|----------|-------------|---------------|-------------------|
| 設定難易度 | 最も簡単 | 簡単 | 中〜難 |
| セキュリティ | 中（Synologyリレー経由） | 高（WireGuard暗号化） | 低（直接公開） |
| 速度 | リレー時遅い/直接時高速 | 高速（P2P） | 最速（直接） |
| ルーター設定 | 不要 | 不要 | 必要（ポート転送） |
| モバイル対応 | Synologyアプリ全対応 | Tailscaleアプリ必要 | ブラウザ/アプリ |
| コスト | 無料 | 無料（個人100台まで） | 無料 |
| 対応サービス | DSM, Drive, Photos等 | NAS全サービス+LAN | NAS全サービス |
| 制限事項 | 大ファイル転送遅い場合あり | DSMのSSH不可・Subnet制限あり | セキュリティリスク高 |

### 3.2 推奨構成: QuickConnect + Tailscale 併用

```
■ 日常利用 → QuickConnect
  - Synology Drive でファイルアクセス
  - DS file / DS photo でモバイルアクセス
  - 設定ゼロ、すぐ使える

■ 高速・高セキュリティ → Tailscale
  - 大ファイル転送時
  - 管理者アクセス（DSM管理画面）
  - LAN内と同等の速度・機能
```

### 3.3 QuickConnect 設定手順

```
1. DSM → コントロールパネル → 外部アクセス → QuickConnect
2. 「QuickConnectを有効にする」にチェック
3. Synologyアカウントでログイン（未登録なら作成）
4. QuickConnect IDを設定（例: tas-nas）
   → アクセスURL: https://QuickConnect.to/tas-nas
5. 「詳細設定」で許可するサービスを選択:
   - Synology Drive ✓
   - File Station ✓
   - DSM ✓
   - Photos ✓
   - 不要なサービスは無効化
6. 適用
```

### 3.4 Tailscale 設定手順

```
1. DSM → パッケージセンター → 「Tailscale」検索 → インストール
2. Tailscaleパッケージを開く → ログイン（Google/Microsoft等）
3. 他デバイス（PC/スマホ）にもTailscaleインストール
4. 同じアカウントでログイン → 自動接続
5. NASのTailscale IP（100.x.x.x）でDSMにアクセス可能

■ サブネットルーター設定（LAN全体アクセス）:
  - DSM → Tailscale → Settings → Subnet routes
  - ローカルネットワーク範囲を追加（例: 192.168.1.0/24）
  - Tailscale管理画面で承認
```

### 3.5 セキュリティ設定（必須）

```
■ DSM セキュリティ強化（QuickConnect有効化前に実施）

1. 2FA（二要素認証）の有効化
   DSM → コントロールパネル → セキュリティ → アカウント
   → 「2要素認証を強制」→ 管理者グループに適用
   → Synology Secure SignIn または TOTP（Google Authenticator等）

2. 管理者アカウント
   - デフォルト「admin」アカウントを無効化
   - 別名の管理者アカウントを使用
   - パスワード: 16文字以上、大小英数記号混在

3. 自動ブロック
   DSM → コントロールパネル → セキュリティ → 保護
   → ログイン試行: 5回失敗で永久ブロック
   → 許可リスト: 自社IPを追加

4. ファイアウォール
   DSM → コントロールパネル → セキュリティ → ファイアウォール
   → 日本国内IPのみ許可（海外アクセス不要なら）

5. SSL証明書
   DSM → コントロールパネル → セキュリティ → 証明書
   → Let's Encrypt無料証明書を取得・適用

6. DSMポート変更
   デフォルト5000/5001 → 別ポートに変更

7. 通知設定
   DSM → コントロールパネル → 通知
   → ログイン通知をメールで受信
```

### 3.6 モバイルアクセス（スマートフォン）

| アプリ | 用途 | QuickConnect | Tailscale |
|--------|------|-------------|-----------|
| Synology Drive | ファイル閲覧・編集 | ○ | ○ |
| DS file | ファイル管理 | ○ | ○ |
| DS photo | 写真閲覧 | ○ | ○ |
| DS cam | 監視カメラ | ○ | ○ |
| Synology Secure SignIn | 2FA認証 | - | - |

```
設定手順（iOS/Android共通）:
1. App Storeから上記アプリをインストール
2. QuickConnect ID（例: tas-nas）でログイン
3. 2FA認証を完了
4. オフラインアクセス: 重要ファイルをピン留め
```

## 5. 運用ルール

1. **暗号化パスワード**: 紛失すると復元不可。パスワードマネージャーに保管
2. **復元テスト**: 四半期に1回、実際にファイルを復元して確認
3. **容量監視**: C2の使用量を月次確認。1TB超えそうなら対象フォルダ見直し
4. **DSMアップデート**: セキュリティパッチは速やかに適用
5. **Tailscaleデバイス管理**: 退職者・紛失デバイスは即座に削除
