### i-Construction 2.0の概要

国土交通省は建設業の生産性向上を目的に「i-Construction」を推進しています。2025年度からは**i-Construction 2.0**へ進化し、BIM/CIMの原則適用拡大、3次元データの施工全プロセスでの活用が標準化されました。

ICT土工とは、**測量・設計・施工・検査の全工程でICT技術を活用する土工事**のことです。従来の断面管理から面的管理へ移行し、施工品質の均一化と検査工数の大幅削減を実現します。

### ICT土工の5ステップ

| ステップ | 工程 | 内容 |
|:---:|------|------|
| 1 | **3次元起工測量** | UAV写真測量またはレーザスキャナにより現況地形の3次元点群データを取得 |
| 2 | **3次元設計データ作成** | 点群データと2次元図面からTINサーフェスモデル（3次元設計データ）を構築 |
| 3 | **ICT建機による施工** | 3次元設計データをGNSS搭載建機に取り込み、マシンコントロール/ガイダンスで施工 |
| 4 | **3次元出来形管理** | 施工完了後にUAV測量で出来形点群を取得し、設計データとの差分を面的に比較 |
| 5 | **3次元データの納品** | 起工測量・設計・出来形の全3次元データと検査記録を電子納品 |

### ドローン測量の位置づけ

ICT土工においてドローン（UAV）測量は、**ステップ1（起工測量）とステップ4（出来形管理）**の両方で中核技術として機能します。

- **広範囲の面的計測**: 従来のトータルステーション（ポイント計測）と異なり、数百万〜数千万点の高密度点群で地表面全体を捕捉
- **工期短縮**: 数ヘクタールの現場でも半日〜1日で撮影完了。従来測量比で作業時間を最大80%削減
- **出来形の面的評価**: 設計面との差分をヒートマップで可視化し、施工精度を面全体で定量評価

> **国交省の方針**: 2026年度以降、直轄土木工事ではICT活用を原則とし、3次元データを活用した施工・検査が標準仕様となっています。

### 出来形管理基準における精度要件

国土交通省「UAVを用いた公共測量マニュアル（案）」および「3次元計測技術を用いた出来形管理要領（土工編）」に基づく精度基準は以下の通りです。

| 項目 | 基準値 | 備考 |
|------|--------|------|
| **計測精度（出来形）** | **±5cm以内** | 設計面との標高差で評価 |
| **水平精度（XY方向）** | ±2〜3cm | RTK/PPK補正適用時 |
| **鉛直精度（Z方向）** | ±3〜5cm | GCP・検証点による補正後 |
| **点群密度** | 1点/m²以上 | 出来形管理に必要な最低密度 |
| **地上画素寸法（GSD）** | 1〜3cm/pixel | 飛行高度・カメラ性能に依存 |

### 使用機材の条件

ICT土工で使用するUAV・計測機材は、以下の要件を満たす必要があります。

**UAV（ドローン）本体**
- GNSS（GPS/GLONASS/BeiDou等）対応のマルチ衛星測位機能
- RTK/PPKによるリアルタイムまたは後処理キネマティック補正に対応
- 飛行ログ（軌跡・姿勢・高度）の記録・出力機能

**カメラ・センサー**
- 有効画素数2,000万画素以上の高解像度カメラ
- メカニカルシャッター搭載（ローリングシャッター歪みの防止）
- キャリブレーション済みレンズ（内部標定要素が既知であること）

**GNSS受信機（GCP設置用）**
- 2周波以上のGNSS受信機（L1/L2対応）
- スタティック測量またはRTK測量に対応
- 日本の電子基準点ネットワーク（GEONET）との整合性

### GCP（地上基準点）設置基準

GCPは3次元計測精度を担保するための最重要要素です。

| 要件 | 基準 |
|------|------|
| **最低設置数** | 外周部に4点以上 + 内部に検証点（チェックポイント）2点以上 |
| **配置間隔** | 撮影範囲の外縁に均等配置。対角線上の配置を推奨 |
| **標識サイズ** | 飛行高度に応じて15cm〜50cm角の対空標識を使用 |
| **測量方法** | GNSS測量（スタティック/RTK）で3級基準点相当以上の精度を確保 |
| **座標系** | 平面直角座標系（日本測地系2011: JGD2011） |
| **成果精度** | 水平・鉛直ともに±2cm以内 |

> **精度検証のポイント**: GCPとは別に設置したチェックポイントで計測精度を第三者検証し、精度検証レポートとして納品します。残差がXY方向±3cm、Z方向±5cm以内であることを確認してください。

### 納品すべきデータ形式

ICT土工の電子納品では、以下のデータを所定の形式で提出します。

| データ種別 | ファイル形式 | 用途・備考 |
|-----------|------------|-----------|
| **点群データ** | LAS / LAZ | XYZ座標+RGB色情報を含む3次元点群。LAZはLASの圧縮形式（容量1/5〜1/10） |
| **オルソ画像** | GeoTIFF（.tif） | 幾何補正済み正射投影画像。GIS・CADで座標付きで利用可能 |
| **DSM（数値表面モデル）** | GeoTIFF（.tif） | 構造物・植生を含む表面標高モデル |
| **DTM（数値地形モデル）** | GeoTIFF（.tif） | フィルタリング処理で地表面のみを抽出した標高モデル |
| **TINサーフェスモデル** | LandXML（.xml） | 3角形メッシュによる設計面・出来形面。ICT建機への入力データとしても使用 |
| **出来形管理図** | PDF / Excel | 設計面との差分ヒートマップ、断面比較図、数量計算書 |
| **精度検証レポート** | PDF | チェックポイントでの残差一覧、精度評価結果 |
| **飛行記録** | CSV / PDF | 飛行日時、飛行経路、気象条件、使用機材情報 |

### 座標系・測地系の要件

| 項目 | 仕様 |
|------|------|
| **測地系** | 日本測地系2011（JGD2011） |
| **座標系** | 平面直角座標系（当該地域の系番号を適用） |
| **標高基準** | 東京湾平均海面（T.P.） |
| **座標参照系（EPSG）** | 例: EPSG:6675（第VII系/東海地方）等、地域に応じて設定 |
| **BIM/CIM連携** | IFCフォーマット対応（発注者指定時） |

### メタデータ要件

電子納品データには以下のメタデータを付与します。

- **計測日時**: 撮影年月日・時刻
- **使用機材**: UAV機体名・カメラ型番・GNSS受信機型番
- **飛行諸元**: 飛行高度・オーバーラップ率（縦80%以上/横60%以上を推奨）・GSD
- **処理ソフトウェア**: SfM/MVSソフト名・バージョン
- **GCP情報**: 設置点数・座標値・測量方法
- **精度情報**: チェックポイント残差（RMSE値）

### 電子納品フォルダ構成（例）

```
SURVEY/
├── WORK/               … 起工測量データ
│   ├── POINTCLOUD/     … 点群データ（LAS/LAZ）
│   ├── ORTHO/          … オルソ画像（GeoTIFF）
│   ├── DSM/            … 数値表面モデル
│   └── GCP/            … 基準点成果・精度検証レポート
├── ASBUILT/            … 出来形データ
│   ├── POINTCLOUD/     … 出来形点群
│   ├── ORTHO/          … 出来形オルソ
│   ├── COMPARISON/     … 設計面との差分比較
│   └── REPORT/         … 出来形管理帳票
└── DESIGN/             … 3次元設計データ
    ├── TIN/            … TINサーフェスモデル（LandXML）
    └── DRAWING/        … 設計図面
```
