# 書類スキャン・デジタル化計画

## 概要
個人+会社（TAS / TOMOSHI）混在の紙書類をデジタル化し、NASに格納する。
フォルダ構成を先に作成し、分類ルールを決めてからスキャン開始する。

---

### 命名規則

```
YYYYMMDD_種類_相手先_概要.pdf

例:
20260115_契約書_三上税理士法人_顧問契約.pdf
20251201_請求書受領_ドットシンク_サーバー保守.pdf
20260301_確定申告_R7年分.pdf
20260310_飛行許可_DIPS_包括申請更新.pdf
```

- 日付: 書類の発行日（不明なら受領日、それも不明なら `00000000`）
- 種類: フォルダ名と同じカテゴリ名を使用
- 相手先: 会社名・機関名（個人書類は省略可）
- 概要: 内容を端的に記述

## 2. スキャン手順

### 推奨機材

| 優先度 | 機種 | 価格帯 | 特徴 |
|--------|------|--------|------|
| 1 | ScanSnap iX1600 | 約4.5万円 | ADF両面40枚/分、Wi-Fi、OCR付属、名刺対応 |
| 2 | ScanSnap iX1300 | 約3万円 | 省スペース、ADF両面30枚/分 |
| 3 | スマホアプリ（Adobe Scan等） | 無料 | 少量の場合の代替手段 |

**推奨: ScanSnap iX1600**（大量書類を一気に処理できる。名刺もA3も対応）

### スキャン設定

| 項目 | 設定値 |
|------|--------|
| 解像度 | 300dpi（標準）/ 契約書・登記等の重要書類は 600dpi |
| カラーモード | 自動判別（カラー/グレー/白黒） |
| ファイル形式 | PDF/A（長期保存向け、ISO規格） |
| 両面 | 有効（両面スキャン） |
| 白紙ページ | 自動削除 |
| 傾き補正 | 有効 |
| 圧縮 | 中（ファイルサイズと画質のバランス） |

### OCR設定

- **ScanSnap Home内蔵OCR**: スキャン時に自動実行（日本語+英語）
- 検索可能PDFとして保存
- OCR精度確認: 重要書類は手動で文字検索テスト
- 手書き文字が多い書類: OCR非適用、ファイル名で検索性を担保

### スキャンワークフロー

```
1. 書類を「個人 / TAS / TOMOSHI / 不明」に物理仕分け
2. ステープル・クリップ除去
3. ADF投入 → スキャン（00_INBOX に保存）
4. ファイル名を命名規則に従いリネーム
5. 該当フォルダに移動
6. 紙原本の処理（後述）
```

## 3. 分類ルール

### 判定フローチャート

```
書類を手に取る
  ├─ 宛先が「東海エアサービス」「TAS」→ 02_TAS/
  ├─ 宛先が「TOMOSHI」「灯し」→ 03_TOMOSHI/
  ├─ 宛先が「國本洋輔」「豊田洋輔」（個人名のみ）
  │   ├─ 内容が事業関連 → 02_TAS/ or 03_TOMOSHI/
  │   └─ 内容が私的 → 01_個人/
  └─ 判断つかない → 00_INBOX/（後で判断）
```

## 4. デジタル化後の紙の扱い

### 法定保存期間

| 書類種別 | 保存期間 | 根拠法 |
|----------|----------|--------|
| 法人税関連（帳簿・決算書） | **7年**（欠損金あり: 10年） | 法人税法 |
| 消費税関連（請求書・領収書） | **7年**（インボイス制度対応） | 消費税法 |
| 給与関連（源泉徴収簿等） | **7年** | 所得税法 |
| 社会保険関連 | **2年** | 健保法・厚年法 |
| 労働保険関連 | **3年** | 労基法 |
| 契約書 | **契約終了後5〜10年** | 商法・民法 |
| 定款・登記関連 | **永久** | — |
| 許認可証 | **有効期間中+更新後3年** | — |
| 個人の確定申告 | **5年**（7年推奨） | 所得税法 |
| 医療費領収書 | **5年**（医療費控除用） | 所得税法 |
| 保険証券 | **契約期間中+解約後5年** | — |

### 廃棄ルール

1. **スキャン完了 + ファイル名確認 + OCR検索テスト** → 3点完了で紙は廃棄可能
2. ただし以下は **紙原本を保管**（スキャン不可ではなく、原本も残す）:
   - 定款・登記簿謄本（原本必要な場面がある）
   - 契約書（署名・押印原本）
   - 許認可証
   - 不動産関連（権利証等）
   - 保険証券
3. 廃棄前に `99_廃棄待ち/YYYY/` にスキャン済みであることを記録
4. **シュレッダー処理**（個人情報・取引情報を含むため）
5. 廃棄判断に迷うものは保管（コストはほぼゼロ）

### 紙原本の保管方法（残すもの）

```
ファイルボックス（個人用）: 1箱
ファイルボックス（TAS用）:  1箱
ファイルボックス（TOMOSHI用）: 必要になったら追加

→ NASと同じフォルダ構成のインデックスタブを貼る
```

## 5. 実行スケジュール

| ステップ | 作業内容 | 所要時間目安 |
|----------|----------|-------------|
| 1 | NASにフォルダ構成作成 | 15分 |
| 2 | 書類の物理仕分け（個人/TAS/TOMOSHI/不明） | 1〜2時間 |
| 3 | スキャナー準備・設定 | 30分 |
| 4 | スキャン実行 | 書類量次第（100枚/30分目安） |
| 5 | ファイル名リネーム・フォルダ移動 | スキャン枚数 x 30秒 |
| 6 | OCR検索テスト（重要書類） | 30分 |
| 7 | 紙原本の仕分け（保管/廃棄） | 1時間 |
| 8 | シュレッダー処理 | 30分 |

## 6. 注意事項

- **電子帳簿保存法**: 2024年1月以降の電子取引データは電子保存が義務。紙で受領した書類のスキャナ保存は任意だが、保存要件（タイムスタンプ or 訂正削除履歴）を満たせば紙廃棄可能
- **freee請求書**: 既にデジタル発行（INV形式）のためスキャン不要。受領側の紙請求書が対象
- **改姓関連**: 豊田→國本の改姓に伴う書類（戸籍、各種届出控え）は `01_個人/身分証明/` に集約
- **NAS障害対策**: RAID1以上 or クラウドバックアップ併用を推奨
