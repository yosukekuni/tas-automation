# Claude Code Agent Teams設計

作成日: 2026-03-14 / ステータス: 凍結（設計書のみ）

## 概要

複数のClaude Codeセッションがタスクを共有・ブロードキャストし、24/365の自律稼働体制を構築する。Mac mini常駐エージェントとの統合を含む。

## アーキテクチャ

```
┌─────────────────────────────────────────────┐
│              Lark Base (タスクHub)            │
│  tblGrFhJrAyYYWbV (タスク管理)               │
│  tblIyLVn7RFqDbdt (会話ログ)                 │
└──────┬──────────┬──────────┬────────────────┘
       │          │          │
  ┌────▼───┐ ┌───▼────┐ ┌──▼─────────┐
  │Session A│ │Session B│ │Mac mini常駐 │
  │(WSL)    │ │(MacBook)│ │(M4/32GB)    │
  └────────┘ └────────┘ └────────────┘
```

### タスク共有メカニズム

1. **Lark Baseをハブとしたポーリング方式**
   - 各セッションが起動時・定期的にLark Baseから未アサインタスクを取得
   - タスクの「担当セッション」フィールドで排他制御
   - 完了時にステータス更新 + 会話ログ記録

2. **ブロードキャスト（Lark Webhook）**
   - 緊急タスク発生時: Lark Webhookで全セッションに通知
   - セッション障害時: watchdogが検知しタスクを再アサイン

### Mac mini常駐との統合

- Mac miniエージェント（project_mac_mini_agent.md参照）が常時稼働
- GitHub Actions定期実行の補完（15分間隔の隙間を埋める）
- 重い処理（点群処理、大量メール送信等）はMac miniに優先アサイン

### セッション管理

```python
# agent_team_manager.py
# - セッション登録/解除
# - ハートビート（5分間隔）
# - タスクアサイン（優先度×セッション負荷）
# - 障害検知・タスク再アサイン（ハートビート3回連続ミスで障害判定）
```

## 運用ルール

- **タスク競合防止**: Lark Baseのレコードロックでアサイン排他制御
- **セッション識別**: `{デバイス名}_{PID}_{起動時刻}` で一意識別
- **ログ集約**: 全セッションの会話ログをtblIyLVn7RFqDbdtに集約

## 前提条件

- Mac mini M4/32GB 購入済み（or 購入予定）
- Claude Code複数セッション同時実行の安定性確認
- Lark Base APIレートリミット内での運用（5 req/sec）
