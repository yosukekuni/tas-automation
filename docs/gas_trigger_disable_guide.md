# Google Apps Script「無題のプロジェクト」トリガー無効化手順書

作成日: 2026-03-14
関連: [不審メール調査レポート](suspicious_email_report.md)

---

## 背景

2026-03-13、info@tokaiair.com から yosuke.toyoda@gmail.com 宛に意図しないファイル共有メールが送信された。調査の結果、Google Apps Script（GAS）の「無題のプロジェクト」に存在する `shareFileByEmail` 関数がトリガーにより自動実行され、Google Driveのファイル共有メールを送信していたと判断された。実行ログには「Exceeded maximum execution time」エラーも記録されている。

本手順書では、このトリガーの無効化からプロジェクト削除、事後確認までの一連の対応手順を記載する。

---

## 1. Google Apps Script へのアクセス

1. ブラウザで info@tokaiair.com（または該当するGoogle Workspaceアカウント）にログインする
2. https://script.google.com にアクセスする
3. 左メニューの「マイプロジェクト」をクリックし、プロジェクト一覧を表示する

> **注意**: GASプロジェクトはGoogle Driveのファイルとして保存されている。Google Driveからも `https://drive.google.com` で「Apps Script」と検索すると見つかる場合がある。

---

## 2.「無題のプロジェクト」の特定

### 方法A: script.google.com から探す

1. https://script.google.com のプロジェクト一覧で「無題のプロジェクト」を探す
2. 一覧が多い場合、上部の検索バーで「無題」と入力して絞り込む
3. 該当プロジェクトをクリックして開く

### 方法B: Google Drive から探す

1. https://drive.google.com を開く
2. 検索バーに `type:script 無題` と入力して検索する
3. 検索結果から「無題のプロジェクト」を見つけ、ダブルクリックしてGASエディタで開く

### 特定の確認ポイント

プロジェクトを開いたら以下を確認し、正しいプロジェクトであることを確かめる:

- コードエディタに `shareFileByEmail` 関数が存在するか
- 左メニュー「実行数」をクリックし、実行ログに 2026-03-13 付近の実行記録があるか
- 実行ログに「Exceeded maximum execution time」エラーが記録されているか

---

## 3. トリガーの確認・無効化

### 3-1. トリガー一覧の表示

1. GASエディタの左メニューで、時計アイコンの「トリガー」をクリックする
2. 現在設定されている全トリガーの一覧が表示される

### 3-2. トリガーの内容確認

各トリガーについて以下の情報を記録する（削除前の証跡として）:

| 確認項目 | 記録内容 |
|----------|----------|
| 関数名 | （例: shareFileByEmail） |
| イベントのソース | （例: 時間主導型） |
| トリガーの種類 | （例: 特定の日時、分ベース等） |
| 実行間隔 | （例: 1時間ごと） |
| 最終実行日時 | （実行ログから確認） |
| エラーの有無 | （実行ログから確認） |

### 3-3. トリガーの削除（無効化）

1. 削除したいトリガーの右端にある「三点メニュー（︙）」をクリックする
2. 「トリガーを削除」を選択する
3. 確認ダイアログが表示されたら「削除」をクリックする
4. 全てのトリガーに対してこの操作を繰り返す
5. トリガー一覧が空になったことを確認する

> **重要**: トリガーを削除しても、コード自体は残る。手動実行は引き続き可能なため、プロジェクト自体の削除も検討する（セクション4参照）。

---

## 4. 不要なGASプロジェクトの削除

トリガー無効化後、プロジェクト自体が不要であれば削除する。

### 4-1. 削除前のバックアップ（任意）

1. GASエディタでコードファイル（.gs）の内容をコピーし、テキストファイルとして保存する
2. 後日調査が必要になる可能性があるため、コード内容の保存を推奨する

### 4-2. GASエディタからの削除

1. GASエディタの左メニューで「概要」（iアイコン）をクリックする
2. 右上のゴミ箱アイコン、またはページ下部の「削除」オプションを探す

> GASエディタ内に直接の削除ボタンがない場合は、Google Driveから削除する。

### 4-3. Google Drive からの削除

1. https://drive.google.com を開く
2. 検索バーに `type:script 無題` と入力して該当ファイルを見つける
3. ファイルを右クリックし、「削除」を選択する
4. 左メニューの「ゴミ箱」を開き、該当ファイルが入っていることを確認する
5. 完全削除する場合は、ゴミ箱内でファイルを右クリックし「完全に削除」を選択する

> **注意**: ゴミ箱に入れただけではトリガーが停止しない場合がある。必ずセクション3でトリガーを先に削除してから、プロジェクトを削除すること。

## 5. Google Drive の共有設定確認

不審メールで共有されたファイルの権限を確認・修正する。

### 5-1. 該当ファイルの特定

1. https://drive.google.com を開く
2. 不審メールに記載されていた情報を手がかりに検索する:
   - ファイルサイズ: 2047MB
   - 日付: 2022/06/15 - 2022/06/18
   - キーワード: 「工藤」「工事」
3. 左メニューの「共有アイテム」や「最近使用したアイテム」も確認する

### 5-2. 共有設定の確認・修正

1. 該当ファイル（またはフォルダ）を右クリックし、「共有」→「共有」を選択する
2. 共有相手の一覧を確認する
3. 意図しない共有先がある場合:
   - 該当ユーザーの右側のプルダウンで「削除」を選択する
   - または「アクセスを削除」をクリックする
4. 「一般的なアクセス」が「リンクを知っている全員」になっている場合:
   - 「制限付き」に変更し、「完了」をクリックする

### 5-3. Drive全体の外部共有状況の確認

1. https://drive.google.com で検索バーに `is:shared-external` と入力する
2. 外部ユーザーと共有されているファイル一覧が表示される
3. 意図しない外部共有がないか確認する

## 6. info@tokaiair.com からの外部送信確認

同様のメールが外部の第三者に送信されていないか確認する。

### 6-1. Gmail送信履歴の確認

1. info@tokaiair.com で Gmail（https://mail.google.com）にログインする
2. 左メニューの「送信済み」をクリックする
3. 検索バーに以下を入力して検索する:

```
from:info@tokaiair.com subject:こちらに添付 after:2026/03/01
```

4. 該当するメールが他の宛先にも送信されていないか確認する

5. 追加で以下の検索も実施する:

```
from:info@tokaiair.com subject:ファイル共有 after:2026/03/01
```

```
from:info@tokaiair.com subject:アップロード after:2026/03/01
```

```
from:info@tokaiair.com 工藤 after:2026/03/01
```

### 6-2. GAS実行ログからの送信先確認

1. GASエディタ（プロジェクト削除前に実施）で左メニューの「実行数」をクリックする
2. 実行ログを確認し、`MailApp.sendEmail` や `GmailApp.sendEmail` の呼び出し先アドレスを特定する
3. 自社以外のアドレスに送信された記録がないか確認する

### 6-3. Google Workspace 管理コンソールからの確認（管理者のみ）

1. https://admin.google.com にログインする
2. 「レポート」→「監査と調査」→「メールログ検索」を選択する
3. 送信者に `info@tokaiair.com` を指定し、期間を 2026-03-01 以降に設定して検索する
4. 送信先が外部ドメインのメールがないか確認する

> **注意**: メールログ検索はGoogle Workspace管理者権限が必要。また、ログの保持期間は最大30日間。

## 再発防止

- GASプロジェクトには必ず識別可能な名前を付ける（「無題のプロジェクト」のまま放置しない）
- トリガーを設定する場合は、エラー通知を有効にする（GASトリガー設定の「通知設定」）
- 定期的に https://script.google.com で不要なプロジェクト・トリガーがないか棚卸しする
- メール送信を伴うGASには、送信先のホワイトリストチェックをコード内に実装する
