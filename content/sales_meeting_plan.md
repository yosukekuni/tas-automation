# 営業定期MTG 運用計画

## 概要

- 対象: 國本（CEO）、新美、政木
- 頻度: **週次**（状況安定後は隔週に移行可）
- 所要時間: 30分厳守
- 目的: パイプラインの可視化、ステージ進捗の加速、受注率の改善

---

| 時間 | セクション | 内容 | 担当 |

| 0-5分 | 数字レビュー | 週次KPI確認（下記参照） | 國本 |
| 5-15分 | パイプライン確認 | Hot/Warm案件の個別レビュー、ネクストアクション確認 | 全員 |
| 15-20分 | 学び共有 | 今週うまくいったこと・断られた理由 | 新美/政木 |
| 20-25分 | 課題・ブロッカー | 進まない案件の相談、國本の介入判断 | 全員 |

- 報告は事前にCRM更新で済ませる。MTG中の「報告」は禁止
- 議論は「次に何をするか」に集中する
- 30分で終わらない案件は別途1on1で対応

## 3. KPIダッシュボード定義

### 週次追跡指標

| # | 指標 | 定義 | データソース | 目標値（週） |
|---|------|------|-------------|-------------|
| 1 | 新規商談数 | 今週新規作成された商談レコード数 | 商談テーブル（作成日） | 新美: 8件、政木: 5件 |
| 2 | ステージ進捗数 | ステージが前進した商談数 | 商談テーブル（ステージ変更） | 各3件以上 |
| 3 | 見積提出数 | 「見積提出」ステージに移行した件数 | 商談テーブル | 各2件以上 |
| 4 | 受注数 | 今週受注した件数 | 受注台帳 | 合計1件/週 |
| 5 | 受注金額 | 今週受注した合計金額 | 受注台帳 | ¥100万/週 |
| 6 | Hot案件数 | 温度=Hot の商談数 | 商談テーブル | 常時5件以上 |
| 7 | 次アクション設定率 | 次アクションが設定されている商談の割合 | 商談テーブル | 100% |
| 8 | ステージ設定率 | ステージが設定されている商談の割合 | 商談テーブル | 100% |

### 目標値の根拠

現状（6ヶ月実績）:
- 241商談 → 受注1件 = 受注率0.4%
- ヒアリング→見積変換率: 0%

**まず達成すべき最低ライン**:
- 受注率: 0.4% → **3%**（業界平均の下限）
- 見積提出率: 0% → **10%**（月に5件は見積を出す）
- 月間受注: ¥450万/6ヶ月 = ¥75万/月 → **¥200万/月**（固定費¥69万をカバー+利益）

これを週次に分解:
- 週13件の商談活動 × 10%見積 = 月5件見積
- 月5件見積 × 40%受注率 = 月2件受注
- 月2件 × ¥100万/件 = ¥200万/月

### 月次追跡指標（参考）

| 指標 | 目標 |
|------|------|
| 月間受注金額 | ¥200万以上 |
| 見積提出数 | 5件以上 |
| 受注率（見積→受注） | 40%以上 |
| CRMデータ充足率 | ステージ・次アクション100% |
| 新規取引先開拓数 | 5社以上 |

## 4. MTG運用ルール

### 基本設定

| 項目 | 内容 |
|------|------|
| 頻度 | **毎週** |
| 曜日 | **月曜 10:00-10:30** |
| 場所 | Larkビデオ通話 or 対面 |
| 欠席ルール | 前日までに連絡。欠席でもCRM更新は必須 |

**月曜10:00の理由**: 週の頭に数字を見て行動計画を立てる。朝一は避け、メール確認後に開始。

### 事前準備（金曜中 or 月曜9:30まで）

**新美・政木がやること**:
1. 今週活動した商談のステージを更新する
2. 全Hot/Warm案件の「次アクション」を記入する
3. 新規商談があれば商談報告フォームから登録する

**國本がやること**:
1. CRMからKPI数値を抽出（自動化済み: weekly_sales_kpi.py）
2. ステージ未設定・次アクション未設定をチェック（自動化済み: lark_crm_monitor.py）
3. 未更新案件があれば月曜朝に本人へLark通知

### フォローアップ

| タイミング | アクション | 担当 |
|-----------|-----------|------|
| MTG直後 | 議事録をLarkチャットに共有 | 記録者 |
| 水曜 | アクションアイテムの中間チェック（Larkメッセージ） | 國本 |
| 金曜 | CRM更新リマインド（自動通知） | Bot |

### 段階的な運用改善

**Phase 1（今すぐ〜1ヶ月）**: 週次MTG定着
- 毎週必ず開催する。内容の質より「やる」ことを優先
- CRM更新の習慣づけ（未更新→通知）
- ステージ設定率を100%にする

**Phase 2（1〜2ヶ月後）**: 質の向上
- 見積提出件数を追い始める
- 受注率の改善施策を議論
- 國本のクロージング介入ルールを明確化

**Phase 3（3ヶ月後〜）**: 自走化
- KPIが安定したら隔週に移行
- 議事録・KPI抽出の自動化を進める
- 1on1（國本×各メンバー）を月1で追加

## 5. 國本の介入基準

以下に該当する案件は、MTGで報告の上、國本がクロージングに入る:

| 条件 | アクション |
|------|-----------|
| 見積金額 ¥100万以上 | 國本が同行 or 電話フォロー |
| 先方の意思決定者が出てきた | 國本から直接連絡 |
| 2回以上見積を出して決まらない | 國本が値引き・条件交渉 |
| 競合が出ている | 國本が差別化ポイントを説明 |
| 担当が「もう無理」と感じた | 國本がリカバリーを試みる |

## 6. CRM運用との連携

### MTGまでに必ず埋めるフィールド

商談テーブル（tbl1rM86nAw9l3bP）:
- **ステージ**: 必須。未設定は許容しない
- **温度（Hot/Warm/Cold）**: 必須
- **次アクション**: 必須。空欄=放置=機会損失
- **次アクション日**: 必須

### 自動チェック（既存スクリプト活用）

- `lark_crm_monitor.py`: 15分毎にCRM監視、未設定フィールドを検出→Lark通知
- `weekly_sales_kpi.py`: 月曜朝にKPIレポート自動生成
- `auto_followup_email.py`: Hot/Warm案件のフォローメール自動生成

## 付録: 最初のMTGでやること

初回MTGは通常より10分延長（40分）し、以下を実施:

1. **この運用ルールの共有**（5分）
   - 目的: 数字を見て、次の行動を決める場
   - ルール: 報告はCRMで済ませる、MTGは議論の場

2. **現状の数字を全員で直視する**（5分）
   - 241商談→受注1件の事実
   - 月¥69万の固定費が出ている事実
   - 危機感の共有（責める場ではない）

3. **各自のHot/Warm案件を棚卸し**（15分）
   - 今すぐ動かせる案件はどれか
   - ステージ未設定の商談を全て埋める

4. **今週の具体的アクション決定**（10分）
   - 各自3件、今週中に必ず進める案件を選ぶ
   - 具体的な行動（電話する/訪問する/見積出す）と期限

5. **次回MTG日時確定**（5分）
